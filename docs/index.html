Most engines walk the syntax tree on every render. Veloz resolves paths, binds filters and lays out control flow at compile time. What runs at request time is compact bytecode on a pooled stack VM.

Syntax you already know

If you have written Twig, Jinja or Blade, you already write Veloz. Inheritance, filters, a real loop context and expressions that read like expressions.

invoice.tpl
{% extends "layout" %}
{% block body %}
{% for line in invoice.lines %}
  {{ loop.index }}. {{ line.concept | capitalize }}
     {{ line.total() | money }}
{% endfor %}
Status: {{ invoice.paid ? "PAID" : "PENDING" }}
Client: {{ client ?: "guest" }}
{% endblock %}
main.go
engine := veloz.New()

engine.RegisterFilter("money",
    func(args []veloz.Value) (veloz.Value, error) {
        return veloz.Str(format(args[0])), nil
    })

tmpl, err := engine.Compile("invoice", src)
out, err := tmpl.Render(map[string]any{
    "invoice": inv,
    "client":  "Estudio Mares",
})

Everything on board

Syntax

Twig-style expressions

Ternary, elvis ?:, in, string concat ~, power **, negative indexing, array and map literals.

Composition

Template inheritance

extends, block with defaults and overrides, plus include for partials. Cycles detected at render.

Iteration

Real loop context

loop.index, first, last, revindex, length, key-value maps and a for/else branch for empty lists.

Library

25+ built-in filters

From capitalize to sort, map and date. Register your own with one call; unknown names fail at compile time with line and column.

Safety

Autoescape by default

HTML output is escaped unless you mark it raw or return a SafeString. Opt out per engine when generating non-HTML.

Runtime

Pooled, concurrent-safe

Interpreters, stacks and locals are pooled and reused. Zero fixed allocations per render, safe across goroutines.
